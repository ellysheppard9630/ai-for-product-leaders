# PRD Assistant Workflow

A workflow for using AI to support Product Requirements Documents.

## Inputs

- Business objective
- Customer problem
- Target users
- Scope
- Constraints
- Success metrics

## Workflow

1. Summarize the customer problem
2. Define goals and non-goals
3. Identify user stories
4. Draft acceptance criteria
5. Highlight open questions
6. Suggest risks and dependencies

## Output

A structured PRD draft that can be reviewed, edited, and validated with stakeholders.
