# Discovery to PRD Example

## Scenario

A product team has identified that users struggle to understand which tasks require immediate attention.

## Discovery Notes

- Users report switching between multiple screens to understand priorities.
- Managers want a faster way to identify exceptions.
- Current workflows rely heavily on manual review.

## AI-Assisted PRD Output

### Problem Statement

Users need a faster way to identify priority tasks so they can focus on the work that requires immediate action.

### Goals

- Reduce time spent reviewing routine items
- Highlight exceptions clearly
- Support faster decision-making

### Non-Goals

- Replace existing workflow tools
- Automate final business decisions

### User Stories

- GIVEN I am reviewing my work queue, WHEN priority items exist, THEN I can clearly identify which items require attention.
- GIVEN I am a manager, WHEN I open the dashboard, THEN I can see exceptions that may need follow-up.

### Success Metrics

- Reduced time to identify priority tasks
- Increased usage of dashboard filters
- Improved user satisfaction with task visibility
