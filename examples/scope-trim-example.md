# Scope Trim and MVP Example

## Scenario

A team is planning a dashboard enhancement, but the initial scope is too large for the release window.

## Original Scope

- Custom dashboard widgets
- Role-based dashboard layouts
- Export to PDF
- Real-time alerts
- Saved filters
- Historical trend charts

## MVP Recommendation

Focus the MVP on the smallest set of capabilities that solve the core user problem.

### Keep

- Saved filters
- Basic priority indicators
- Simple dashboard layout

### Defer

- Custom widgets
- PDF export
- Historical trend charts
- Real-time alerts

## Rationale

The MVP should help users identify important work faster without requiring a full dashboard redesign.

## Risks

- Users may still request deeper customization
- Deferred reporting features may need a follow-up release
- Stakeholders may need alignment on what “MVP” means

## Recommended Next Step

Validate the trimmed scope with stakeholders and confirm whether saved filters and priority indicators are enough to support the first release.
